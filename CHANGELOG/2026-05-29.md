## Bug Fixes

- **Keycloak AppHost config** (`c03d396`) — Removed the `IsRunMode` conditional block that called
  `.WithDataVolume()` and a duplicate `.WithRealmImport()` on the Keycloak resource. The duplicate
  import caused realm duplication errors on startup; the data volume caused stale realm files to
  persist across sessions. Keycloak lifetime changed to `Session` and `--verbose` flag added.

- **Realm filename case mismatch** (`a8ac169`) — Renamed `toten-realm.json` → `ToTen-realm.json`.
  Keycloak's `DirImportProvider` enforces that the filename matches the `"realm"` key value
  exactly (case-sensitive). Forced via `git mv -f` to bypass macOS case-insensitive filesystem.
  Also erased drifted EF migration chain: `InitialCreate` had been rewritten after its first
  apply, leaving the database with an orphaned `Items` table. All migrations deleted; fresh
  single-migration `InitialCreate` generated from the current 15-table model snapshot.

- **Keycloak health check stall** (`760b2f7`) — `AddKeycloak` auto-registers a
  `HealthCheckAnnotation` against the HTTPS management port (9000). That port uses an
  Aspire-generated self-signed cert (`CN=localhost`) rejected by the AppHost's `HttpClient`,
  so `WaitFor(keycloak)` stalled indefinitely. Removed the annotation via
  `ResourceAnnotationCollection.Remove` and replaced with
  `WithHttpHealthCheck("/realms/master")` on HTTP port 8080, which returns 200 once Keycloak
  has finished startup and realm initialisation.

- **PostGIS extension missing** (`a5cf0be`) — The default `postgres:17.6` image has no PostGIS
  shared libraries, so the EF migration's `CREATE EXTENSION IF NOT EXISTS postgis` failed.
  Switched `RunAsContainer` to `postgis/postgis:17-3.4` (PostgreSQL 17 + PostGIS 3.4). Tag
  `17-3.5` has no `linux/arm64` manifest on Docker Hub; `17-3.4` is the highest available
  arm64 tag.

- **Minimal API DI binding error** (`e50ee0f`) — `IEventPublisher` injected into a Minimal API
  endpoint handler was being read from the request body by the framework (not allowed in
  Minimal APIs). Added `[FromServices]` attribute to explicitly bind from DI container.

## Implementation Notes

### EF Core Migration Drift Diagnosis
When `dotnet ef` reports `relation "X" does not exist`, connect directly to the running Postgres
container and compare `\dt` output against `__EFMigrationsHistory`. A mismatch means a migration
was rewritten after apply. Recovery: delete all migration files and the snapshot, drop/recreate
the data volume, regenerate with `dotnet ef migrations add InitialCreate --output-dir Data/Migrations`.

### Aspire Keycloak Health Check Internals
`Aspire.Hosting.Keycloak` (13.x) wires a `HealthCheckAnnotation` against the `management`
endpoint (HTTPS/9000) using a self-signed cert that the AppHost process does not trust.
`ResourceAnnotationCollection` is `IList<IResourceAnnotation>` — use `OfType<HealthCheckAnnotation>().ToList()`
then `Remove(item)` (no `RemoveAll` method exists). Substitute with `/realms/master` on the
`http` endpoint.

### Docker Volume / Image Lifecycle
`ContainerLifetime.Session` does **not** clean up named Docker volumes — volumes are independent
of container lifecycle. Always pair `Session` lifetime with no `WithDataVolume()` call if the
intent is ephemeral state. When switching a container image, remove the existing volume explicitly;
data directories from different base images are not always compatible.

### arm64 Image Availability
Before pinning a Docker image tag in `WithImageTag`, verify the manifest includes
`linux/arm64/v8` — `docker pull <image>:<tag>` will exit non-zero with a manifest error on
Apple Silicon if the platform is absent. For PostGIS, `17-3.4` is the current arm64 ceiling.
